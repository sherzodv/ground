# Golden tests

Principles:

- Grouped by syntax
- Ordered from simple to advanced
- Minimal and focused
- Comprehensive but not redundant

When changing or adding new conform to the principles above:

- Find the right place to add/change things: classify first, do proper naming
- Keep the test minimal, only describe minimum code to verify the tested functionality
- Use proper multiline strings
- Cover all cases and edge cases for the tested functionality
